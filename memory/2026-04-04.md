## 经验反思（可复用思维逻辑）

### ✅ 成功的策略

1. **浏览器自动化不要死磕 CDP**
   - 之前疯狂扫描 CDP 端口、启动 Edge/Chrome 调试模式，全部失败（端口被 aDrive 占用、Chrome 合并进程、Playwright 连接超时）
   - 真正能跑通的方式：
     - `start https://mp.weixin.qq.com` → 用户手动打开
     - CDP 端口需要先确认 Chrome 是否以 `--remote-debugging-port` 启动
     - **用华为云镜像替代 GitHub 下载** — 国内 GitHub 直连会被 Connection Reset

2. **SkillHub 搜索策略**
   - 搜微信相关 skill 没有后台数据抓取工具（微信不开放 API）
   - 搜浏览器自动化（browser）能找到 browser-use-local
   - 当用户说"太复杂了"时，走 SkillHub 搜索是最快的路径

3. **安装 Node.js 全局包超时问题**
   - `npm install -g` 容易超时（网络慢或依赖复杂）
   - 解决：换镜像 `--registry=https://registry.npmmirror.com`，或用官方源 `--registry=https://registry.npmjs.org`
   - 设置 `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1` 可跳过浏览器内核下载

4. **图片压缩标准流程**
   - PNG 2048x2048 → JPEG 1024x1024, quality=85, optimize=True
   - 缩小 1/4 面积，体积从 2.76MB 降至约 100-200KB
   - 缩放到桌面：`copy` 命令直接复制即可

### ❌ 犯下的错误与陷阱

1. **不要在 Windows cmd 里直接执行多行 Python 代码（`python -c`）**
   - cmd 无法正确处理缩进，会报 IndentationError
   - 正确做法：写成 `.py` 文件，再 `python script.py`

2. **不要假设 `browser_use` 函数可用**
   - 当前 CoPaw agent 环境中 `browser_use` 作为函数调用返回 FunctionNotFoundError
   - 需要通过 `execute_shell_command` 或 CDP 连接来控制浏览器
   - SKILL.md 里有文档不代表函数已注册

3. **下载 Git 不要用 GitHub 直链**
   - `curl -L` 从 GitHub Releases 下载会被 `Connection reset`（exit code 35）
   - 华为云镜像 `https://mirrors.huaweicloud.com/git-for-windows/` 可用

4. **启动 Chrome 调试模式不要用 `start chrome`**
   - `start` 命令会启动浏览器但不会等它加载，且新实例可能合并到已有 Chrome
   - 要用完整路径 `"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir="%TEMP%\chrome-copaw-debug" --new-window`
   - 且需要 `--user-data-dir` 指向空目录，否则会合并到默认 Chrome

5. **`npm install -g browser-use` 下载几百 MB 的内核会超时**
   - 应该加 `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1` 跳过

### 💡 行动指南

- **下载国内资源优先用镜像**：华为云、npmmirror 等
- **Windows 下 Python 脚本一律写文件执行**，别用 `-c` 多行
- **浏览器控制：如果 user 的 Chrome 已经登录了，别试图控制它的窗口**——走数据导出路线更稳
- **SkillHub 是捷径**：用户觉得方案复杂时，优先搜 SkillHub
- **Git 安装后需要处理 PATH**：当前会话通过临时 `set PATH` 解决
